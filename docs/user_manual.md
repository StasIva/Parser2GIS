# Руководство пользователя Parser2GIS

## О программе

Parser2GIS — это программа для сбора данных об организациях из справочника 2GIS. Она позволяет автоматизировать поиск, сбор и экспорт информации о компаниях по заданным городам и рубрикам.

Программа работает в двух режимах: **графический интерфейс** (рекомендуется для повседневной работы) и **командная строка** (для автоматизации и интеграции).

## Начало работы

### Запуск программы

**Графический интерфейс:**
```bash
parser2gis gui
```

**Командная строка:**
```bash
parser2gis --help
```

При первом запуске автоматически создаются:
- Директория `~/.parser2gis/` с файлом базы данных
- Директория `~/.parser2gis/logs/` для логов
- Таблицы в SQLite (автомиграция)
- Тестовые города (Москва, Санкт-Петербург, Новосибирск) и рубрики

## Графический интерфейс

### Главное окно

Главное окно содержит:
- **Панель инструментов** — кнопки основных действий
- **Таблица задач** — список всех созданных задач
- **Строка состояния** — количество задач и статус

### Кнопки панели инструментов

| Кнопка | Описание |
|--------|----------|
| Создать | Создать новую задачу сбора данных |
| Старт | Запустить выбранную задачу |
| Стоп | Остановить выполняющуюся задачу |
| Экспорт | Экспортировать данные |
| Обновить справочник | Обновить список городов и рубрик из 2GIS |
| Настройки | Открыть настройки приложения |

### Создание задачи

1. Нажмите **Создать** на панели инструментов
2. Выберите город из списка (доступны города из справочника)
3. Выберите рубрику из дерева категорий
4. Введите название задачи (или оставьте авто-сгенерированное)
5. Нажмите **ОК**

### Управление задачами

- **Запуск** — выберите задачу в таблице и нажмите **Старт**
- **Остановка** — выберите выполняющуюся задачу и нажмите **Стоп**
- Статус задачи отображается в колонке **Статус**: создана, выполняется, приостановлена, завершена, ошибка
- Прогресс отображается в процентах (0–100%)

### Экспорт данных

1. Нажмите **Экспорт**
2. Выберите задачу для экспорта
3. Выберите формат: XLSX, CSV или JSON
4. Укажите путь для сохранения
5. Нажмите **Экспортировать**

### Обновление справочника

1. Нажмите **Обновить справочник**
2. Программа загрузит актуальный список городов и рубрик из 2GIS
3. Результат отобразится в диалоговом окне

### Настройки

Доступные настройки:
- **Задержка между запросами (мс)** — пауза между запросами к API 2GIS
- **Количество потоков** — число параллельных потоков при сборе данных
- **Путь к данным** — директория для хранения базы данных
- **Настройки прокси** — HTTP/HTTPS прокси для подключения

## Командная строка

### parser2gis run

Запускает задачу сбора данных из командной строки:

```bash
parser2gis run --city "Москва" --rubric "Автосервис" --output results.xlsx
```

Параметры:
- `--city` — название города (обязательно)
- `--rubric` — название рубрики (обязательно)
- `--output` — путь к файлу для автоматического экспорта

### parser2gis list

Просмотр списка ресурсов:

```bash
parser2gis list cities     # список городов
parser2gis list rubrics    # список рубрик
parser2gis list tasks      # список задач
```

### parser2gis export

Экспорт результатов задачи:

```bash
parser2gis export --task-id 1 --format xlsx --output report.xlsx
```

Параметры:
- `--task-id` — ID задачи (обязательно)
- `--format` — формат: `xlsx`, `csv` или `json` (по умолчанию xlsx)
- `--output` — путь к выходному файлу (обязательно)

### parser2gis update-directory

Обновление справочника городов и рубрик:

```bash
parser2gis update-directory --resource all      # города и рубрики
parser2gis update-directory --resource cities   # только города
parser2gis update-directory --resource rubrics  # только рубрики
```

### parser2gis seed

Заполнение базы тестовыми данными (3 города, 10 рубрик):

```bash
parser2gis seed
```

### parser2gis gui

Запуск графического интерфейса:

```bash
parser2gis gui
```

### chatgpt-export

Экспорт диалога ChatGPT в Markdown:

```bash
chatgpt-export --export-path conversations.json --conversation-id <ID> --output chat.md
chatgpt-export --export-path conversations.json --conversation-id <ID> --output handoff.md --handoff
```

## Сборка и установка

### Установка из исходного кода

```bash
git clone https://github.com/IvaStas/parser2gis.git
cd parser2gis
python -m venv .venv
source .venv/bin/activate
pip install -e .
```

### Сборка portable-версии

```bash
pip install -e ".[build]"
python scripts/build.py
```

Готовая сборка будет в директории `dist/parser2gis/`. Для Windows дополнительно настраивается метаинформация (версия, иконка, манифест).

## Данные и логи

### База данных

- **Путь:** `~/.parser2gis/parser2gis.db`
- **Формат:** SQLite (WAL-режим)
- **Автомиграция:** таблицы создаются и обновляются автоматически
- **Полнотекстовый поиск:** поиск по названиям организаций
- **Резервное копирование:** автоматически перед удалением данных
- **Проверка целостности:** при каждом запуске

### Логи

- **Путь:** `~/.parser2gis/logs/`
- **Ротация:** каждый лог-файл не более 5 МБ, до 3 архивных копий
- **app.log** — общие события приложения
- **tasks/** — отдельные файлы для каждой задачи
- **errors.log** — ошибки источника данных (2GIS)
- **exports.log** — история экспортов

## Часто задаваемые вопросы

### Как добавить новый город в справочник?

Нажмите **Обновить справочник** или выполните:
```bash
parser2gis update-directory
```

### В каком формате можно выгрузить данные?

XLSX, CSV (UTF-8 с BOM) и JSON.

### Где хранятся данные?

В `~/.parser2gis/parser2gis.db`. Можно изменить путь в настройках.

### Как сбросить базу данных?

Удалите файл `~/.parser2gis/parser2gis.db` и выполните `parser2gis seed`.
